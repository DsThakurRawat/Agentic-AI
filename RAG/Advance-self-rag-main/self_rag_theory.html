Self-Reflective RAG (Self-RAG)

A masterclass on building autonomous, self-correcting RAG systems. Explore adaptive retrieval gating, semantic relevance filtering, fact-grounding checks (IsSUP), strict revision loops, and query rewriting mechanics.

Module 1: The RAG Trilemma & Hallucination Vectors

Traditional RAG pipelines assume that blindly piping retrieved documents into an LLM context window guarantees factuality. In reality, this linear paradigm creates three massive failure modes:

  • ❌ Unnecessary Retrieval: Querying database storage for queries that could be resolved via parametric knowledge, adding latency and cost.
  • ❌ Irrelevant Ingestion: Injecting semantic noise from poorly chunked contexts, which distracts the LLM and degrades synthesis quality.
  • ❌ Abstraction Hallucination: Falsely representing details or generating qualitative statements that are unsupported by the original facts.

Linear RAG Risk Vector

Linear flows offer no route to verify factuality or correct abstractions before sending the response to the user.

Module 2: Self-RAG - The Self-Correcting State Machine

**Self-RAG** solves this trilemma by structuring RAG as a state machine governed by self-reflection steps. Instead of executing in a single straight line, a Self-RAG agent actively queries itself:

1. Adaptive Gating

Evaluates whether external knowledge is actually needed. If the model can answer using its own memory, it skips database lookup entirely.

2. Relevance Filter

Validates retrieved pages. It throws away irrelevant chunks to ensure only high-signal text feeds into the prompt context.

3. Grounding (IsSUP)

Verifies that the generated response does not introduce any abstract claims or qualitative terms unsupported by the source text.

4. Query Refiner Loop

If an answer fails usefulness checks, the agent rewrites the query, pulls fresh documents, and retries up to three times.

Module 3: Step-by-Step State Node Mechanics

Step 1-3: Routing & relevance Gating

When a user runs a query, `decide_retrieval` evaluates if specific facts are needed. If `should_retrieve` is `True`, it invokes semantic retrieval; otherwise, it answers from direct memory.

Once documents are retrieved, `is_relevant` grades them at a topic level. If local database results are completely irrelevant, the graph branches to `rewrite_query` and initiates a live **Tavily Web Search** fallback.

Step 4-5: Fact Grounding (IsSUP) & Revision Loop

Generating an answer isn't the final step. The **IsSUP** check validates grounding. If the LLM generates an answer that is `partially_supported` or `no_support` (e.g., calling policies "generous" or "employee-first" when the text only lists basic dates), the node routes to `revise_answer`.

The **Strict Reviser** rewrites the answer using **quote-only** bullet points direct from the text and loops back to IsSUP. A state variable `retries` bounds the loop (max 10 retries) to prevent infinite loops.

The Blended Refinement Paradigm

If an answer is grounded but graded as `not_useful` by the usefulness checker (`IsUSE`), the graph invokes a **query rewriting loop**. The query is stripped of conversational fluff, injected with 2-5 high-signal search keywords, and loops back to retrieval. This guarantees recovery even in highly ambiguous cases!

LangGraph TypedDict State Schema

from typing import List, TypedDict, Literal
from langchain_core.documents import Document

class State(TypedDict):
    question: str
    retrieval_query: str
    rewrite_tries: int
    need_retrieval: bool
    docs: List[Document]
    relevant_docs: List[Document]
    context: str
    answer: str
    issup: Literal["fully_supported", "partially_supported", "no_support"]
    evidence: List[str]
    retries: int
    isuse: Literal["useful", "not_useful"]
    use_reason: str

IsSUP Fact Grounding Prompts

class IsSUPDecision(BaseModel):
    issup: Literal["fully_supported", "partially_supported", "no_support"]
    evidence: List[str] = Field(default_factory=list, description="Direct quotes from context.")

issup_prompt = ChatPromptTemplate.from_messages([

               "Return JSON with keys: issup, evidence.\n"
               "issup must be: fully_supported, partially_supported, or no_support.\n\n"
               "Rules:\n"
               "- Be strict: if the answer contains ANY qualitative embellishment not explicitly stated,\n"
               "  grade as partially_supported.\n"
               "- Do not use external knowledge."),

])

Strict Quote-Only Revision Node

revise_prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a STRICT reviser.\n\n"
               "You must output based on the following format:\n\n"
               "FORMAT (quote-only answer):\n"
               "- <direct quote from the CONTEXT>\n"
               "- <direct quote from the CONTEXT>\n\n"
               "Rules:\n"
               "- Use ONLY the CONTEXT.\n"
               "- Do NOT add any new words besides bullet dashes and the quotes themselves."),
    ("human", "Question:\n{question}\n\nCurrent Answer:\n{answer}\n\nCONTEXT:\n{context}")
])

def revise_answer(state: State):
    out = llm.invoke(revise_prompt.format_messages(
        question=state["question"],
        answer=state["answer"],
        context=state["context"]
    ))
    return {
        "answer": out.content,
        "retries": state.get("retries", 0) + 1
    }

LangGraph Flow Construction

g = StateGraph(State)

# Nodes
g.add_node("decide_retrieval", decide_retrieval)
g.add_node("generate_direct", generate_direct)
g.add_node("retrieve", retrieve)
g.add_node("is_relevant", is_relevant)
g.add_node("generate_from_context", generate_from_context)
g.add_node("is_sup", is_sup)
g.add_node("revise_answer", revise_answer)
g.add_node("is_use", is_use)
g.add_node("rewrite_question", rewrite_question)

# Flow edges
g.add_edge(START, "decide_retrieval")
g.add_conditional_edges("decide_retrieval", route_after_decide, {
    "generate_direct": "generate_direct", 
    "retrieve": "retrieve"
})
g.add_edge("retrieve", "is_relevant")
g.add_conditional_edges("is_relevant", route_after_relevance, {
    "generate_from_context": "generate_from_context",
    "no_answer_found": "no_answer_found"
})
g.add_edge("generate_from_context", "is_sup")
g.add_conditional_edges("is_sup", route_after_issup, {
    "accept_answer": "is_use",
    "revise_answer": "revise_answer"
})
g.add_edge("revise_answer", "is_sup")
g.add_conditional_edges("is_use", route_after_isuse, {
    "END": END,
    "rewrite_question": "rewrite_question",
    "no_answer_found": "no_answer_found"
})
g.add_edge("rewrite_question", "retrieve")

app = g.compile()
