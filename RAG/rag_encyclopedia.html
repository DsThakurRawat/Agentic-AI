Section 1: Parametric vs Non-Parametric Knowledge

Traditional LLMs depend entirely on **Parametric Knowledge**—meaning the facts stored in their static weights computed during pre-training. This limits their context window and causes hallucinations when queries require real-time updates.

"RAG establishes a Non-Parametric memory lane, providing target vector context directly inside the dynamic prompt."

By translating text chunks into spatial coordinate tensors (embeddings), semantic search retrieves context corresponding closely to the user's inquiry, preventing generic pre-trained responses.

Naive Vector Database Retrieval

from langchain_community.vectorstores import FAISS
from langchain_openai import OpenAIEmbeddings

# Initialize local FAISS vector index
embeddings = OpenAIEmbeddings()
db = FAISS.from_documents(chunks, embeddings)

# Retrieve k nearest neighbors
retriever = db.as_retriever(search_kwargs={"k": 2})

Recursive Character Text Splitting

    separators=["\n\n", "\n", " ", ""]
