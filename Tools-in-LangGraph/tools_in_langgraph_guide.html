Section 1: The Cyclic Function-Calling Pattern

Tools are the interfaces that allow an LLM or Agent to interact with the outside world. Instead of just generating text, an agent with tools can search the web, execute code, query a database, or call an external API.

When an agent uses tools in LangGraph, it follows a strict cyclic pattern:

  • Agent Decision Node – Evaluates the chat state. If it needs facts beyond its weights, it generates a structured tool_call list detailing targets and arguments.
  • Tool Execution Node – Automatically runs the local python function using the parsed JSON parameters, returning a strict ToolMessage.
  • Consensus Generation – The model digests the observations inside the system prompt and synthesizes a highly grounded final response.

Custom Decorator API Setup

from langchain_core.tools import tool

@tool
def get_user_data(user_id: int) -> dict:
    """
    Fetches the profile details for a specific user ID.
    Always invoke this tool when you need names, emails, or roles.

    Args:
        user_id: The unique integer profile ID of the user.
    """
    db = {
        1: {"name": "Alice", "role": "Admin"},
        2: {"name": "Bob", "role": "Developer"}
    }
    return db.get(user_id, {"error": "Profile not found."})

Pydantic Argument Enforcement

from pydantic import BaseModel, Field

class WeatherSchema(BaseModel):
    city: str = Field(description="Name of the city, e.g. London")
    unit: str = Field(default="celsius", description="fahrenheit or celsius")

@tool(args_schema=WeatherSchema)
def get_weather(city: str, unit: str = "celsius") -> str:
    """Fetch live weather metrics for a specified city."""
    return f"The weather in {city} is 22 degrees {unit}."

LLM Binding & ToolNode Orchestration

from langgraph.prebuilt import ToolNode
from langchain_openai import ChatOpenAI

tools = [get_user_data, get_weather]

# Initialize LLM & Bind Tools
llm = ChatOpenAI(model="gpt-4o")
llm_with_tools = llm.bind_tools(tools)

# ToolNode processes structured calls
tool_node = ToolNode(tools)
