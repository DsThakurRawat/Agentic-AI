CORE LLM CONSTRAINT

Context Overflow Theory

Why "Unlimited Memory" is physically impossible in current Transformer architectures.

1. The Tokenization Layer

LLMs do not read words; they read **Tokens**. A token is a numerical representation of a chunk of characters. On average, 1000 tokens ≈ 750 words.

In Short-Term Memory, every turn of the conversation adds more tokens to the "History". This is a linear growth: O(N) where N is the number of messages.

2. The Quadratic Scaling Problem

Most LLMs use the Self-Attention Mechanism. To understand the context, every token looks at every other token in the sequence. This means the computational cost grows quadratically (N²).

Cost = (Tokens)2

Because of this math, providers (OpenAI, Anthropic) set a hard limit called the Context Window to prevent their servers from melting down.

3. The Context Window Equation

The total capacity is shared between your **Input** and the model's **Output**. If your input (STM) is too large, there is no "room" left for the model to think.

  • Prompt_Tokens (Your instructions)
  • History_Tokens (The STM messages)
  • Completion_Tokens (The AI's actual response)
Total ≤ Max_Context_Limit

For example, if GPT-4o has a 128k limit, but you send 127.9k tokens of history, the model can only generate a few words before it "hits the wall" and cuts off.

4. Symptom: The "Lost in the Middle" Effect

Even before an overflow error happens, long context causes performance issues. Studies show that LLMs are great at remembering the Start and End of a prompt, but they often forget the middle. This is why trimming and summarization are not just for saving space—they are for Accuracy.
