Section 1: The Central Persistent Store Paradigm

Unlike short-term thread partitioning (which isolates conversation histories inside specific thread tokens), **Long-Term Memory (LTM)** leverages a centralized database schema to save preferences, concepts, and metadata cross-session.

"LTM allows the agent to recognize the user's name, coding preferences, and projects even if the user starts a fresh thread."

This is executed via structural reflection runs: during a conversation, memory nodes parse chat lines, extract structured user configurations, and write them to a persistent Postgres table.
