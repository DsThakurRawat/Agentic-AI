Safe Concurrency &
Custom Aggregation

Advanced graph runtimes require cumulative state operations. **Custom State Reducers** guide the framework engine on how to safely merge concurrent updates returned by independent parallel nodes.

The Problem: Value Clobbering

In standard state schemas, if two parallel nodes return updates to the same key, the framework uses "Last Write Wins" logic. One update is silently overwritten (clobbered).

graph TD A[Start] --> B[Node 1: score=8] A --> C[Node 2: score=5] B --> D{Race} C --> D D --> E[Final State: score=5] style E fill:#450a0a,stroke:#991b1b

The Solution: State Reducers

By annotating keys with reducer functions (like `operator.add`), we tell the engine to merge values instead of replacing them. Lists are concatenated safely.

graph TD A[Start] --> B[Node 1: [8]] A --> C[Node 2: [5]] B --> D[Reducer] C --> D D --> E[Final State: [8, 5]] style E fill:#0c4a6e,stroke:#0ea5e9

The Annotated Pattern

To implement safe parallel appending, we use the `Annotated` type from Python's typing library, passing a reduction function as metadata. 

import operator
from typing import Annotated, TypedDict

class ParallelState(TypedDict):
    # operator.add triggers list concatenation
    audits: Annotated[list[str], operator.add]

Custom Reduction Logic

You aren't limited to `operator.add`. You can author custom functions to handle complex merging (e.g., taking the max, averaging, or merging dictionaries).

def custom_sum(current: int, new: int):
    return current + new

class State(TypedDict):
    throughput: Annotated[int, custom_sum]
