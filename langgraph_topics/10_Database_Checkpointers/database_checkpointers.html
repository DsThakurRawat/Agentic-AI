01 The Case for Persistence

Why Databases?

In production environments, `MemorySaver` is insufficient because process restarts wipe all state. By using relational databases, we ensure:

  • Durability: State survives server crashes and deployments.
  • Scalability: Multiple application instances can share a single database backend.
  • Observability: Database tables can be queried directly for auditing and debugging.

Backend Architectures

SqliteSaver

Ideal for local development and edge apps. Serializes superstep execution payloads directly into flat `.db` files. 

from langgraph.checkpoint.sqlite import SqliteSaver

with SqliteSaver.from_conn_string("checkpoints.db") as checkpointer:
    app = graph.compile(checkpointer=checkpointer)

AsyncPostgresSaver

Production-grade. Implements asynchronous connection pools and JSONB indexing for high-throughput microservices. 

from langgraph.checkpoint.postgres import AsyncPostgresSaver

pool = ConnectionPool(conninfo)
checkpointer = AsyncPostgresSaver(pool)
await checkpointer.setup()
