01 Theoretical Foundations

The Tracing Hierarchy

A **Trace** represents the complete end-to-end execution. It is structured as a tree of **Spans**:

  • Root Span: The top-level `graph.invoke()` call.
  • Child Spans: Nodes, LLM calls, Tools, and Retrievers.

State Mutation Visibility

Unique to LangGraph, observability allows you to see exactly how the `State` evolved after each node. Compare **Inbound State** with **Outbound Update** to verify logic flow and debug complex overwrites.

Code Implementation

import os
# Global toggle for the tracing engine
os.environ["LANGCHAIN_TRACING_V2"] = "true"

def node(state):
    # This call is automatically traced
    llm = ChatOpenAI(model="gpt-4o")
    return {"messages": [llm.invoke(state["messages"])]}
# Pass runtime metadata for advanced filtering
config = {
    "configurable": {"thread_id": "user_42"},
    "tags": ["production", "high-priority"],
    "metadata": {"customer_tier": "platinum"}
}

app.invoke(input, config=config)
