01 Theoretical Foundations

The Role of Persistence

HITL is only possible when the graph is **Persistent**. When an interrupt is triggered, the engine serializes the current `State` and instruction pointer into a database. The thread stays "suspended" until a resume command is received.

Resuming with Command()

To "jump-start" a paused execution, developers use the `Command` object. This can pass a simple approval ("yes") or a full state override to "fix" a model's faulty decision before it proceeds.

Interruption Strategies

interrupt_before: Static checkpoint configured during compilation. Pauses *before* a node is invoked.
interrupt_after: Pauses *after* a node completes. Useful for auditing generated outputs.
interrupt(): Dynamic pause triggered inside a node or tool based on runtime ambiguity.
