01 Theoretical Foundations

The Case for Modularity

As systems grow, monolithic graphs become unmanageable. Subgraphs allow you to isolate internal logic (e.g., retrieval, scoring) and expose only a clean interface to the parent orchestrator.

  • Separation of Concerns: Hide internal node complexity.
  • Unit Testing: Test sub-graphs in isolation.

State Interaction Models

Independent State: Subgraph has its own `TypedDict`. Requires manual mapping in a bridge node.
Shared State: Both graphs share the same data schema. Updates are automatically reflected globally.

Code Implementation

# 1. Compile Subgraph Independently
subgraph_builder = StateGraph(SubState)
subgraph_builder.add_node("translate", translate_node)
subgraph_builder.add_edge(START, "translate")
subgraph = subgraph_builder.compile()
def bridge_node(state: ParentState):
    # Map Parent State -> Subgraph State
    sub_res = subgraph.invoke({"input_text": state["eng_draft"]})
    
    # Map Subgraph Result -> Parent State
    return {"hin_draft": sub_res["translated_text"]}
