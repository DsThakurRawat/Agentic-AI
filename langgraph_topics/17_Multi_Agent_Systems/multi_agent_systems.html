Multi-Agent Systems (MAS) represent the peak of agentic complexity. Decompose monolithic problems into specialized, autonomous agents coordinated by a central LLM Supervisor.

01 The Multi-Agent Dichotomy

Supervisor-Worker Pattern

A central orchestrator (the Supervisor) receives the user query and determines the optimal routing.

  • Supervisor: LLM node selecting the next worker.
  • Workers: Specialized nodes (Researcher, Coder, Editor).

Peer-to-Peer Collaboration

Agents operate on a shared state without a central supervisor. They pass the execution thread based on pre-defined conditional edges or scalar triggers.

State Unity

All agents must adhere to a Unified Data Contract to ensure seamless handoffs.
