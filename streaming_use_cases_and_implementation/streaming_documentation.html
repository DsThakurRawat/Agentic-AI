Section 1: The Principle of Real-Time Generation

In traditional request-response architectures, the inference gateway blocks client connections until the model compiles the entire response payload. This causes a massive wait time (**Time-to-First-Token**) of up to 10 seconds.

"Streaming solves this by yielding token deltas incrementally as they are computed, lowering TTFT under 200 milliseconds."

This is achieved using persistent network tunnels that push data using Server-Sent Events (SSE) or bidirectional gRPC protocol channels.

OpenAI SSE Architecture

Server-Sent Events (SSE) provide a unidirectional push method via text/event-streams. It relies on standard REST architecture, making it highly simple to integrate in standard browsers.

• Lightweight Unidirectional Push
• Terminates on standard [DONE] key

FastAPI Server Sent Events endpoint

from fastapi import FastAPI
from fastapi.responses import StreamingResponse
import asyncio

app = FastAPI()

async def token_generator():
    tokens = ["Lang", "Graph", " ", "is", " ", "extremely", " ", "powerful."]
    for token in tokens:
        yield f"data: {token}\n\n"
        await asyncio.sleep(0.1)
    yield "data: [DONE]\n\n"

@app.get("/stream")
async def stream_tokens():
    return StreamingResponse(token_generator(), media_type="text/event-stream")
